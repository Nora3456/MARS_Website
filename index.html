M.A.R.S

Medical AI for Remote Space-Care

OPERATION

User Interface

The system will be easy to use, including simple controls that allow crew members to easily and quickly access information.

Output

Given the data, the system will create diagnostic insights, providing possible diagnoses and next step recommendations.

Autonomous

The system will run autonomously. Blackouts, limited memory, and power won’t be an issue.

Input

Data from biosensors, wearable devices, imaging systems, and crew reports will be collected and saved in the system.

CORE MAJOR FUNCTIONS

  • Data Collection: Gather patient symptoms and history.
  • Machine Learning: Add training data that is well vetted.
  • AI Processing: Analyze input using a machine learning model.
  • Recommendation Engine: Suggest diagnoses or next steps.
  • User Interface: Allows users to interact easily.
  • Keyboard
  • Audio
  • Video
  • Real-Time Data Feeds
  • Non-Real-Time Data Feeds

POSSIBLE MEDICAL EMERGENCIES

Due to the extreme environmental factors of space, astronauts are more likely to experience certain medical emergencies and conditions.

Having a “doctor” on board can help astronauts monitor and predict possible medical situations, such as:

Psychological Changes

  • Anxiety
  • PTSD
  • Depression
  • Other psychological disorders

Traumatic Injuries

  • Head trauma
  • Injuries after missions
  • Radiation exposure

Why is this important?

Symptoms that are detected and input into the system aid the AI in creating possible diagnoses that can ultimately save the life of an astronaut.
